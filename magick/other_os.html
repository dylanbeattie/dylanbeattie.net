Windows PerlMagick How-to: Non-Win2K Operating Systems

You probably also want to check out Ron Savage's Perl CGI Image::Magick FAQ

Windows XP:

According to Michael Chan and several others, these instructions work on Windows XP with no modifications.

Windows NT 4:

  • The Image-Magick.ppd file and the corresponding x86 directory both have to be in the C:\ directory, not C:\TEMP, or it fails. (The ppm program, at some point, decides to look for "\.\x86\Image-Magick.tar.gz", which because of the leading backslash is NOT relative to the "current" directory.)
  • The CORE*.dll files, all of which MUST be placed in C:\perl\bin or the program fails. The IM*.dll files MUST be placed
    in the C:\imagemagick directory (apparently, whatever the value of %MAGICK_HOME% was set to). It didn't seem to matter where the X*.dll files were placed.

Thanks to John Freed for these tips.

Windows 98 SE

  • Instructions as for Windows 2000, but be sure to put the PATH and MAGICK_HOME settings in your autoexec.bat file so they're re-loaded on boot-up.

Thanks to Gerhard Petrowitsch for this tip.

Apache

Eelco Weijmans writes:

"I'd like to add one thing for Win98/Apache users: Besides adding IM to the Path and adding the MAGICK_HOME var, one should really not forget to add the line "PassEnv MAGICK_HOME" to httpd.conf. Took me a while to find this out, and accidently saw it in one of the posts in the mailing list."

I suspect this applies to all Windows versions of Apache and not just Win98.
